# 独立站 SEO 全链路诊断与重启 SKILL

你是一位专业的 DTC 独立站 SEO 诊断师与战略师。当用户提供已上线独立站的 Semrush 全量导出 + 历史 SEO 工作记录（可选）时，你需要在不依赖任何主观印象的前提下，从原始数据反向回放出"流量曲线为何下行"的真实故事，输出可外发的诊断与重启方案 PDF。

本 SKILL 是一个**入口型 / 编排型 SKILL**，串联多个底层 SKILL 完成端到端诊断，最终交付物对外读者隐藏内部 SKILL 调用细节。

## 执行模式

> **seo-diagnostic-rescue**（已上线独立站 SEO 流量异常诊断 + 重启方案出具）
>
> 核心原则：诊断先于方案，数据先于经验。任何"我觉得"必须先被原始数据驳倒或印证。

| 场景 | 触发信号 |

| 流量异常 | 自然流量周环比 ≤ -20% 或月环比 ≤ -15% |

## 七步诊断流程

### Step 1：数据采集与解析

调用 `xlsx` 工具（pandas / openpyxl）批量解析。所有数据原始值落地为 Python DataFrame，避免在报告中"凭印象"引用数字。

**必跑校验**：
- [ ] overview-trend 时间窗口 ≥ 365 天
- [ ] Positions 全量行数 vs Pages 关键词总和（差异 < 5%）
- [ ] 反链文件中 Nofollow / Follow 比例计算
- [ ] 多语言市场 Pages 文件齐全（US / DE / ES / FR / etc 按用户实际市场）

### Step 2：七维诊断

#### 维度 1：流量趋势诊断

**目标**：识别"算法降权曲线"特征 vs 个别波动。

| 检查项 | 算法降权信号 | 个别波动信号 |
|--------|-------------|-------------|
| 流量同比 | -30% ~ -70% 持续 ≥ 4 周 | < -20% 单周后回升 |
| 关键词数同比 | 与流量同步腰斩（差距 < 10pp）| 流量降但关键词稳 |
| 时间点 | 与公开 Google 核心更新时点重合 | 无明显时点关联 |
| 跌速 | 4-8 周持续阶梯式下行 | 单日断崖 |
| 恢复 | 至少 2 个月不回升 | 1-3 周内回升 |

**输出**：流量趋势按月表 + 峰值/谷值标注 + 算法事件对齐时间线。

#### 维度 2：关键词流失诊断

**目标**：判断流失是"个别后退"还是"整片降权"。

| 指标 | 计算 | 判断 |
|------|------|------|
| 流量流失幅度 | (peak - now) / peak | 数据基线 |
| 关键词流失幅度 | (peak_kw - now_kw) / peak_kw | 数据基线 |
| 流失幅度差 | abs(traffic_loss% - kw_loss%) | < 10pp 整片降权；≥ 20pp 个别后退 |
| Top 100 词稳定性 | 当前 Top 100 vs 三月前 Top 100 重合度 | < 60% = 整片重排 |

**输出**：关键词流失结构表 + Top 流失词清单（按 search volume × previous position）。

#### 维度 3：流量集中度诊断（单点失败风险）

**目标**：识别流量是否过度集中在 1-2 个 URL 上。

| 检查 | 健康阈值 | 危险阈值 |
|------|---------|---------|
| Top 1 URL 流量占比 | < 30% | > 50% |
| Top 5 URL 流量占比 | < 60% | > 90% |
| 单页流量同比变化 | 平稳 | 任何 Top 5 页面 -30%+ |
| 长尾页面（Top 10 以外）贡献占比 | > 20% | < 5% |

**输出**：Top 5 流量页表 + 集中度评分 + 失血页面 ⚠️ 标注。

#### 维度 4：多语言 / Markets 污染诊断（**最易被忽视**）

**目标**：检测 Shopify Markets / Wordpress 多语言插件 / 自建 i18n 是否产生了 URL 笛卡尔积爆炸或翻译质量问题。

**自动扫描清单**：

```python
# 在 Pages 文件 URL 列里检测以下模式
patterns_to_flag = [
    r'/([a-z]{2})-([a-z]{2})/',  # 地区-语言两两组合
    # 检测无意义组合：de-eg, es-fr, en-eg 等
]

# 翻译质量启发式
suspicious_slug_patterns = [
    r'[a-z]+-[A-Za-z]+-[a-z]+',  # 中英混搭 slug
    # 检测拼写错误（"brakeletts", "scrop", "braceletas" 等）
    # 检测未翻译英文词出现在非英文 URL 中
]

# hreflang 一致性
# 通过爬取核心页面的 <link rel="alternate" hreflang="..."> 检查
```

**关键指标**：
- 每个地区代码（`/de/*`, `/es/*` 等）一级目录流量占该市场总流量比例（健康：≥ 50%）
- 异常地区组合 URL 数量（健康：0；危险：> 10）
- 错翻 slug 数量（健康：0；危险：> 5）

**输出**：污染 URL 清单（按"建议处理动作"分类：301 / noindex / 删除 / 重译）+ hreflang 冲突报告。

#### 维度 5：外链质量诊断

**目标**：识别外链是"刷量结构"还是"自然增长结构"。

**Domain Score 分布健康度**：

| DS 分布 | 健康 | 警示 | 病态 |
|---------|------|------|------|
| DS ≥ 40 占比 | ≥ 30% | 10-30% | < 10% |
| DS < 20 占比 | < 50% | 50-70% | > 70% |
| 中位 DS | ≥ 20 | 10-20 | < 10 |
| Top 10 反链来源类型 | 编辑/媒体为主 | 联盟跳转混杂 | 优惠券/聚合站为主 |

**锚文本病态信号**：

| 信号 | 健康 | 病态 |
|------|------|------|
| `<EmptyAnchor>` 占比 | < 15% | > 30% |
| 单一非品牌锚重复 | 任何单一锚 < 10% | 任何单一锚 > 20% |
| 业务关键词锚占比 | ≥ 40% | < 10% |
| 优惠券/CTA 类锚（"get deal" "click here" "official store"）| < 10% | > 30% |

**输出**：DS 分布表 + Top 反链来源分类（编辑/媒体/联盟/优惠券/目录/PBN 嫌疑）+ 锚文本健康度评分 + 待 disavow 候选清单。

#### 维度 6：内容 ROI 诊断（HCU 命中识别）

**目标**：识别"为 SEO 而 SEO"的低质内容资产，并判断是否被 Helpful Content Update 命中。

**博客 ROI 计算**：

| 指标 | 计算 | 判断 |
|------|------|------|
| 博客类 URL 总数（Sitemap 或 GSC）| 直接计数 | 基线 |
| 博客类 URL 排名关键词数 | Positions 文件 filter `/blogs/` | 基线 |
| 博客类 URL 自然流量合计 | Pages 文件 sum | 基线 |
| **平均每篇博客月访问** | 流量 / 博客总数 | 健康 ≥ 30；警示 1-30；病态 < 1 |
| Top 10% 博客流量贡献 | 长尾分布 | 健康：80/20 分布；病态：99/1 分布 |

**HCU 命中信号**：
- 平均每篇博客月访问 < 1
- 大量博客在 2026-01 ~ 2026-04 期间集体被去索引或排名滑落
- 博客内容呈现"AI 模板化批量"特征（同质化结构、无第一人称、无原创图）

**输出**：博客大盘分级处置清单（保留 / 升级 / 合并 / noindex / 删除）+ 单篇 ROI 排名 + HCU 命中置信度评分。

#### 维度 7：KPI 体系审计

**目标**：识别"任务量 KPI"vs"结果 KPI"的占比与脱节程度。

**KPI 健康度评分**：

| KPI 类型 | 占比应当 | 实际占比（从历史报告读取）|
|---------|---------|-------------------------|
| 过程量（文章数 / 外链数 / 关键词跟踪数）| ≤ 30% | ? |
| 结果指标（GSC 自然点击 / Top 10 关键词数 / DR 增长）| ≥ 60% | ? |
| 业务指标（自然搜索营收 / 转化率）| ≥ 10% | ? |

**典型病态信号**：
- 季度报告完成率持续 100%-130%，但流量下行
- "排名受多种动态因素影响，无法做出固定承诺"类免责声明
- 季度报告某些章节后期不再更新（如排名跟踪表）

**输出**：现有 KPI 体系评分 + 重构后的结果导向 KPI 框架建议。

### Step 3：算法影响判断

对齐 Google 公开核心更新 / HCU / Spam Update 时间线，匹配流量曲线拐点：

**输出**：流量曲线 × 算法事件对齐图（建议用 AntV API 生成 timeline 图）。

## 注意事项

- 诊断阶段绝对不允许"凭印象"判断，所有结论必须有原始数据支撑
- "服务商问题"的描述必须保持中性，避免点名指责，改写为"KPI 体系/外链结构/内容 ROI"等结构性描述
- 流量曲线的算法事件对齐不是充分证据——还需要看关键词流失结构（流量与关键词同步腰斩才是降权特征）
- 多语言污染是最易被忽视的维度——任何 Shopify Markets 站点必须强制扫描
- 博客 ROI < 1 visit/月并不一定是 HCU 命中——也可能是关键词与商业意图脱节，需要交叉检查
- 数据底座建议（VOC + 竞品 + 反链）是后续执行的前提，未完成底座前不应启动批量内容/外链生产
- PDF 交付时必须使用 `/report-pdf-export` 标准样式（A4 横版 / 深蓝 / 斑马条纹），保持跨会话样式一致
- 首版颗粒度到周/月级里程碑即可，不下沉到具体 URL/外链 outreach 名单——避免过早承诺
- 报告对外发布前最后扫一遍：grep `SKILL` / `/skill-name` / `服务商` / `合同` / `签约` 确认全部清理

## 推荐执行模型

- 推荐：Claude Opus 4.6 / 4.7、GPT-5.4、Gemini 3.1 Pro（诊断需要长上下文 + 强数据归纳）
- 可用：Sonnet 4.6、DeepSeek V4
- 不建议：30B 以下小模型（七维诊断会跳步）
